# 分析报告规则

`yida-analyze-app` 的目标不是完全还原线上逻辑，而是给后续开发一个足够稳定的上下文入口。

## 报告应回答的问题

1. 这个应用有多少页面，分别是什么类型
2. 哪些页面主要承担数据录入
3. 字段结构是否复杂，是否存在明显命名或配置问题
4. 接下来更适合从表单、页面还是 PRD 入手

## 风险信号

- `corpId` 缺失
- 页面数为 0
- 没有任何表单页
- 没有任何自定义页
- 页面标题重复
- 同一页面内字段标签重复
- 大量字段没有识别出类型或 fieldId

## 推荐动作映射

- 结构完整但页面体验差：优先 `yida-custom-page`
- 字段不够：优先 `yida-create-form-page update`
- PRD 与线上脱节：优先 `yida-sync-app`
